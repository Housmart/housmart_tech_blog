Hack Day 2016ハッカソンに参加しました！

2016-02-17 · 64 words · 1 minute read
event
event · hackathon

今週からHousmartで働く松江です！ 先日、Yahoo! Japanさんが主催する日本最大級のハッカソンイベントであるHack Day 2016に参加してきたのでこの場を借りてご紹介します。 作ったもの KizukAIという、飲み会でちょっとした気遣いのできるAIを作りました。 KizukAIは飲み会のテーブルに置くことを想定して作られており、出来ることは以下の3つです。 ビールがなくなったことを検知してビールの追加注文、またはお冷の注文を行う みんなの笑顔を検知して、合いの手を入れたり一緒に笑う 飲み会中の自然な笑顔画像を集めておく 1つ目の機能は、まさしく夢のビールのオートデプロイです！！ 使ったもの 主に以下のデバイス・APIを使用して作りました。 THETA S Clarifaiの画像解析API Docomoの画像認識API THETA Sは、以下のようにドキュメントに書かれているようにAPIが変更され、先代よりもAPIが扱いものになっていました。 RICOH THETA API v2は、Google社のOpen Spherical Camera API Version 1.0（OSC）に準拠しています。 https://developers.theta360.com/ja/docs/v2/api_reference/ ClarifaiはGoogle Cloud Visionのように、画像に写っている物体や状況に関する情報を返してくれるAPIを提供しています。 例えば以下のビール画像であれば”Beer, Glass, Foam, Cold, Drink, Indoor”といった情報が得られます。 泡があること、冷たいこと、室内であることも認識されていて驚きです。 動画にも同等のことができるようなので、そちらも使うと面白いことができそうです。 Clarifaiのデモ ClarifaiのAPIドキュメント 苦労したところ・工夫したところ Clarifaiの前はGoogle Cloud Visionの使用を検討していましたが審査が間に合いませんでした。 Google Cloud Visionは現在プレビュー版を提供している段階で、使用するには申請が必要です。 審査には時間がかかるので、利用を検討する方は早めに申し込んでおくと良いと思います。 ビールの認識精度も上がらずに思考錯誤をしました。 開発会場で撮影されたビール画像は、背景が白いためかオフィスの写真と判定されビールとなかなか判定されませんでした。 ここは背景に暗めの色の布を置くことで判定率が上がりました。 しかし、白い背景を黒でデータ上で塗りつぶしたり、画像からビールジョッキ部分だけの切り抜いただけではビールの判定率はあまり変わりませんでした。 実際の居酒屋でテストしたところ、ビールが持ち上げられて卓上から離れている時間以外はほぼ認識できていたので、暗めの部屋や十分な泡もビールっぽさとして必要なのかもしれません。 また、開発会場内ではWifi使用者が多かったせいか、THETAのWifi転送速度が非常に遅かったため、デモの90秒以内に画像転送できるよう転送するサイズを小さくするといった工夫もしていました。 TEHTAはWifi接続してAPIを叩く必要があるため、操作するデバイスにはLANケーブルなど別のネットワークが必要なことも注意が必要です。 さいごに 今回の作ったものはこちらのレポジトリに置いています。 https://github.com/housmart/kizukai また、幸いにも入賞することができ、CerevoさんのHackeyを頂いたので、これを使ってまた何か作ったらレポートしようと思います！ Yahoo! Japanさん、非常に楽しいイベントをありがとうございました！

カウルTech Blog開始！

2016-02-08 · 38 words · 1 minute read
member
introduction

ブログ始めました。 はじめまして！カウル開発チームの宮永と申します。 本日よりカウルTech Blogを開始いたしました。 サービスの裏側やシステムに関する技術的トピック、開発チームの雰囲気など カウル開発の「空気」のようなモノが伝わればよいかなと思っております。 ※Tech Blogとありますが、Techなことに特化しない回もあると思いますがあしからず。 今回は第１回ということで自己紹介を。 自己紹介 公式的な自己紹介はカウルのTOP（ https://kawlu.com/market ）にありますので、 そちらとは異なるフランクな内容で行きたいと思います。 私、宮永は福岡県北九州の生まれで根っからのサッカー小僧です。 幼稚園、小学校、中学校、高校、大学とサッカーを続け、サッカーに対する情熱は今も冷めておりません。 ただ、東京はサッカーが出来るところも少ないので近年はフットサルがメインです。 弊社の社員が勢揃いした写真です。 開発の話へ。。 このままだとサッカーブログになりそうなので、開発の話に戻ります。 私は新卒（2011年）から5年ほど楽天株式会社でエンジニア、プロデューサとしてシステム開発を担当させていただきました。 学ぶ事が多く楽天では本当に充実した日々を過ごせたと思っております。 好きな言語は・・という話もしたいのですが、本当に好みがなくどれでもやりだすと好きになります。 ですので、今はカウル開発でも使っているRubyは結構好きです。 カウルではCTOという立派な役職を頂いておりますが、エンジニア業務がメインで 今は仕事をしております。 「とにかく世の中に役立つものを創りたい」欲が強く、今カウルの開発に携われている私は なかなかの幸せ者であると思っております。 圧倒的に自由な不動産市場の創造を目指し、やることは多いですが、 日々精進して大きなサービスとしていけるよう頑張ります。 頻繁にMTGを行い、カウルの改善点を洗い出ししています。 最近ハマっていること 弊社のオフィスは渋谷にあるのですが、とにかく飲食店が豊富で毎日昼食に困りません。 最近は社員皆で国際料理のお店巡りが流行ってます。 （ペルー、オーストラリア、ブラジルなどなど渋谷にはとにかく多くの国際料理店が存在します。） 最後に 今回はこのくらいで終わりとさせていただきます。 引き続き、皆様どうぞよろしくお願いします。
